    Introduction

    This API is used for account management. They can only be called from within the company VPC.

    Internal API ALB: http://alligator-1300632963.us-east-2.elb.amazonaws.com

    Verify

    To verify the sms code:

    curl /v1/verify -X POST -d '{"phone":"512-123-4567","code":"4545"}'

    To verify the email code:

    curl /v1/verify -X POST -d '{"email":"you@example.com","code":"4545"}'

    success returns a consumer id and token:

    {
  "id" : "7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923",
  "token" : "96359f9a-7b6a-11ea-9c52-030cfbd1f923"
}

    This endpoint authenticates a user. It is ment to be used with the external "auth" endpoint. The "auth" endpoint can be used to send the code. When verified with "verify", an id and token is returned upon successful authentication.

    HTTP Request

    POST /v1/verify

    JSON Parameters

    Parameter Description
    phone the phone number to be verified
    email the email address to be verified
    code the verification code
    phone or email may be used for authentication, but not both

    Email

    To add an email to an account:

    curl /v1/email -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923","create":"you@example.com"}'

    To delete an email from an account:

    curl /v1/email -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923","delete":"you@example.com"}'

    To list the emails associated with an account:

    curl /v1/email -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923"}'

    success returns a list of emails:

    [
  "you@example.com",
  "me@example.com"
]

    This endpoint reads, adds, or deletes an email from an existing account.

    HTTP Request

    POST /v1/email

    JSON Parameters

    Parameter Description
    id consumer id
    token consumer token
    create the email address to add
    delete the email address to delete

    Phone

    To add a phone number to an account:

    curl /v1/phone -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923","create":"512-123-4567"}'

    To delete a phone number from an account:

    curl /v1/phone -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923","delete":"512-123-4567"}'

    To list the phone numbers associated with an account:

    curl /v1/phone -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923"}'

    success returns a list of phone numbers:

    [
  {
    "phone":"5121234567",
    "country":"1"
  },
  {
    "phone":"5121234568",
    "country":"1"
  }
]

    This endpoint reads, adds, or deletes a phone number from an existing account.

    HTTP Request

    POST /v1/phone

    JSON Parameters

    Parameter Description
    id consumer id
    token consumer token
    create the phone number to add
    delete the phone number to delete

    Unlock

    To send an authentication code via sms:

    curl /v1/unlock -X POST -d '{"phone":"512-123-4567"}'

    To send an authentication code via email:

    curl /v1/unlock -X POST -d '{"email":"you@example.com"}'

    To verify the sms code:

    curl /v1/unlock -X POST -d '{"phone":"512-123-4567","code":"2c0e976e-7b67-11ea-958a-960bfbd1f923"}'

    To verify the email code:

    curl /v1/unlock -X POST -d '{"email":"you@example.com","code":"2c0e976e-7b67-11ea-958a-960bfbd1f923"}'

    This endpoint unlocks an email address or phone number.

    HTTP Request

    POST /v1/unlock

    JSON Parameters

    Parameter Description
    phone the phone number to be verified
    email the email address to be verified
    code the verification code
    phone or email may be used for authentication, but not both

    Consumer

    To update a consumer token:

    curl /v1/consumer -X POST -d '{"id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923","token":"96359f9a-7b6a-11ea-9c52-030cfbd1f923"}'

    success returns the consumer id and new token:

    {
  "id":"7242c2ac-7b6a-11ea-93c5-fd0bfbd1f923",
  "token":"fe15a8fc-7b66-11ea-90d0-920bfbd1f9233"
}

    This endpoint updates a consumer with a new token.

    HTTP Request

    POST /v1/consumer

    JSON Parameters

    Parameter Description
    id consumer id
    token consumer token

    Errors

    Error content follows the same format:

    {
  "err_id":4,
  "message":"code expired",
  "status":"401"
}

    The API uses the following http error codes:

    HTTP Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- You don't have permission based on your parameters.
    404 Not Found -- Something could not be found.
    Error ID Meaning
    1 no code id is found
    2 code does not match
    3 too many code tries
    4 code expired
    6 no phone number was found
    7 too many auth tries on phone
    8 no email address was found
    9 too many auth tries on email
    10 email address locked
    11 email failed to send
    12 phone number locked
    13 phone number invalid
    14 sms failed to send
    15 email address invalid
    16 no consumer found
    17 email address is already linked
    18 phone is already linked
    19 phone is not linked
    20 email address is not linked
    21 email address is not locked
    22 phone is not locked
    23 phone could not be unlocked
    24 email address could not be unlocked
    25 card details invalid
    26 shipping details invalid
    27 no site found
