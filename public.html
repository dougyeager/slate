    Introduction

    This API is used for passwordless authentication for 1-click purchases and general profile data storage

    Auth

    To send an authentication code via sms:

    curl https://api.goknow.me/v1/auth -X POST -d \
 '{"phone":"512-123-4567","site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923"}'

    To send an authentication code via email:

    curl https://api.goknow.me/v1/auth -X POST -d \ 
 '{"email":"you@example.com","site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923"}'

    To verify the sms code:

    curl https://api.goknow.me/v1/auth -X POST -d \
 '{"phone":"512-123-4567","code":"4545"}'

    To verify the email code:

    curl https://api.goknow.me/v1/auth -X POST -d \
 '{"email":"you@example.com","code":"4545"}'

    success returns key/secret pair and profile information:

    {
    "profile": {
        "public_data": "test2",
        "data": "test"
    },
    "secret": "f5f2d688-80ff-11ea-8262-1c3cfbd1f923",
    "phone": true,
    "ship": true,
    "key": "f5f2d28c-80ff-11ea-8262-1c3cfbd1f923",
    "email": true,
    "card": true
}

    This endpoint authenticates a user.

    HTTP Request

    POST https://api.goknow.me/v1/auth

    JSON Parameters

    Parameter Description
    phone the phone number to be verified
    email the email address to be verified
    site_id the site ID that is requesting authentication
    code the verification code
    phone or email may be used for authentication, but not both

    Check Account

    Check the existance of an account phone number:

    curl https://api.goknow.me/v1/check_account -X POST -d \
 '{"phone":"512-123-4567","site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923"}'

    Check the existance of an account email address:

    curl https://api.goknow.me/v1/check_account -X POST -d \
 '{"email":"you@example.com","site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923"}'

    success public profile information:

    {
    "profile_public_data": "test2",
    "profile_data": false,
    "phone": true,
    "ship": true,
    "email": true,
    "card": true
}

    This endpoint checks to see if an account exists.

    HTTP Request

    POST https://api.goknow.me/v1/check_account

    JSON Parameters

    Parameter Description
    phone the phone number to check
    email the email address to check
    site_id the site ID that is requesting authentication
    phone or email may be used for checking an account, but not both

    Create Account

    Create an account:

    curl https://api.goknow.me/v1/create_account -X POST -d \
  '{ \
    "phone": "512-123-4567", \
    "email": "you@example.com", \
    "site_id": "a7ce0404-80fb-11ea-8c17-d63afbd1f923", \
    "details": { \
      "card": { \
        "pan": "4147202412345678", \
        "exp_mo": "06", \
        "exp_yr": "2020", \
        "zip": "78731", \
        "code": "123" \
      }, \
      "profile": { \
        "data": "sensitive info", \
        "public_data": "hello world" \
      }, \
      "ship": { \
        "name": "Doug Yeager", \
        "zip": "78731", \
        "address": "7200 Spurlock Dr." \
      } \
    } \
  }'

    success returns key/secret pair and profile information:

    {
    "secret": "f5f2d688-80ff-11ea-8262-1c3cfbd1f923",
    "key": "f5f2d28c-80ff-11ea-8262-1c3cfbd1f923",
}

    This endpoint checks to see if an account exists.

    HTTP Request

    POST https://api.goknow.me/v1/create_account

    JSON Parameters

    Parameter Required Description
    phone Y the phone number
    email Y the email address
    site_id Y the site ID that is requesting authentication
    details N details object data
    phone or email or both may be used for creating an account

    Account

    To retreive information associated with a user:

    curl https://api.goknow.me/v1/account -X POST -d \
 '{"key":"f5f2d28c-80ff-11ea-8262-1c3cfbd1f923","secret":"f5f2d28c-80ff-11ea-8262-1c3cfbd1f923"}'

    success returns profile information:

    {
    "profile": {
        "public_data": "test2",
        "data": "test"
    },
    "phone": true,
    "ship": true,
    "email": true,
    "card": true
}

    This endpoint retrieves user information.

    HTTP Request

    POST https://api.goknow.me/v1/account

    JSON Parameters

    Parameter Description
    key user profile key
    secret user profile secret

    Save public data

    save profile public data to a phone number:

    curl https://api.goknow.me/v1/auth -X POST -d \
 '{"phone":"512-123-4567","site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923"}'

    save profile public data to an email address:

    curl https://api.goknow.me/v1/auth -X POST -d \ 
 '{ \
    "email":"you@example.com", \
    "site_id":"a7ce0404-80fb-11ea-8c17-d63afbd1f923", \
    "profile_public_data":"hello world" \
  }'

    This endpoint saves profile public data in a user account.

    HTTP Request

    POST https://api.goknow.me/v1/save_profile_public_data

    JSON Parameters

    Parameter Description
    phone the phone number to be verified
    email the email address to be verified
    site_id the site ID that is requesting authentication
    profile_public_data data to save
    phone or email may be used for saving public data, but not both

    Details object

    Attributes

    details object Required Description
    card N card object
    ship N shiping address object
    profile N profile object

    Card object

    Attributes

    card object Required Description
    pan Y card account number
    exp_mo Y 2 digit month for card expiry
    exp_yr Y 4 digit year for card expiry
    zip Y zip code attached to the card
    code Y secret code on the back of the card

    Ship object

    Attributes

    ship object Required Description
    name Y shipping name
    address Y shipping address line
    zip Y shipping zip code

    Profile object

    Attributes

    profile object Required Description
    data N data that is protected by auth
    public_data N data that is accessible to anyone

    Errors

    Error content follows the same format:

    {
  "err_id":4,
  "message":"code expired",
  "status":"401"
}

    The API uses the following http error codes:

    HTTP Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- You don't have permission based on your parameters.
    404 Not Found -- Something could not be found.
    Error ID Meaning
    1 no code id is found
    2 code does not match
    3 too many code tries
    4 code expired
    6 no phone number was found
    7 too many auth tries on phone
    8 no email address was found
    9 too many auth tries on email
    10 email address locked
    11 email failed to send
    12 phone number locked
    13 phone number invalid
    14 sms failed to send
    15 email address invalid
    16 no consumer found
    17 email address is already linked
    18 phone is already linked
    19 phone is not linked
    20 email address is not linked
    21 email address is not locked
    22 phone is not locked
    23 phone could not be unlocked
    24 email address could not be unlocked
    25 card details invalid
    26 shipping details invalid
    27 no site found
    28 email address exists
    29 phone exists
    30 no profile found
    31 card exists
    32 shipping address exists
    33 card is not linked
    34 shipping address is not linked
    35 profile details invalid
    400 invalid request, not specific
    401 unauthorized, not specific
    404 not found, not specific
